## 1. 프로젝트 개요
* **프로젝트명:** 마이링크 (MyLink)
* **목적:** 개발자 및 크리에이터의 다양한 온라인 활동 링크(포트폴리오, 깃허브, 소셜 미디어 등)를 하나의 페이지(고유 URL)로 모아 공유하고 관리할 수 있는 서비스 제공
* **대상:**
  * 프로필 및 포트폴리오를 직관적으로 정리하고자 하는 개발자
  * 다양한 채널을 운영하며 단일 진입점이 필요한 크리에이터
* **사용자:**
  * **어드민(Admin):** 가입 후 자신만의 링크 페이지를 개설하고 프로필과 링크 목록을 관리하는 사용자
  * **방문자 (Visitor):** 공유한 마이링크 주소(URL)를 통해 페이지에 방문하여 링크를 열람하고 클릭하는 사용자

## 2. 디자인 및 기술 환경 구성
* **디자인 시스템:** `shadcn/ui` 기반의 모던 컴포넌트 적용 (다바이스 해상도에 맞춘 완벽한 **반응형 웹** 최우선 개발)
* **인증 및 데이터베이스:**
  * **인증:** Firebase Authentication (구글 소셜 로그인 단일 지원)
  * **DB 설계:** Firebase Firestore 데이터베이스 사용 (사용자 데이터 내 `displayName` 필드 필수 포함 및 링크 데이터를 **서브 컬렉션(Sub-collection)** 으로 구성하는 분리 모델링 사용)

## 3. 핵심 기능 목록
### 3.1 필수 기능 (MVP)
1. 구글 소셜 로그인 기반 가입/인증
2. **닉네임 기반 URL 할당 (`/닉네임` 형태)** (별도 핸들 제외)
3. **프로필 상시 관리 기능 (아바타, `displayName`, 소개글 수정 지원)**
4. 링크 관리 (추가, 삭제, **인라인(Inline) 편집** 기능만 지원 - 순서 변경 제외)
5. **구글 API(`Google Favicon API`) 기반 썸네일 아이콘 연동**
6. 퍼블릭 링크 페이지 (모바일/PC 반응형 최우선)

### 3.2 추후 구현 및 개선 제안 (V2 기능)
1. **[추후 구현] 통계 대시보드 (Analytics):** 누적 방문자 수 및 개별 링크 클릭수 통계 제공
2. **[개선 제안] 개발자/크리에이터 전용 소셜 위젯:** 하단에 GitHub, 개인 블로그, 유튜브 등의 전용 아이콘만 고정 노출하는 영역 제공
3. **[개선 제안] 테마 커스터마이징:** shadcn/ui를 활용하여 컬러 프리셋을 제공하고 사용자가 직접 테마를 선택할 수 있는 기능
4. **[개선 제안] SEO 및 공유 미리보기(OpenGraph):** 사용자 페이지가 공유될 때 노출될 메타데이터 확인 및 설정 기능
5. **[개선 제안] Bio 마크다운 문법 지원:** 자기소개란에 코드 백틱이나 볼드 처리가 가능한 개발자 특화 요소

## 4. 각 기능의 상세 설명

### [필수] 구글 소셜 로그인 및 닉네임 기반 URL
* 복잡한 가입 절차 없이 Firebase 기반의 **구글 계정 단일 로그인**만 사용하여 온보딩 경험을 단순화한다.
* 도메인 뒤에 연결되는 고유 주소를 구성할 때 임의의 영문 핸들(Handle) 체계를 요구하지 않고, 사용자의 **닉네임(`displayName`) 자체를 URL 경로로 활용**한다. (예: `mylink.com/닉네임`)

### [필수] 프로필 관리
* 본인을 나타내는 **프로필 이미지(Avatar)** 를 업로드하고 최적화하여 적용할 수 있다.
* 프로필 화면에 노출되는 **표시 이름(`displayName`)** 과 짧고 명확한 **단문 자기소개(Bio)** 를 언제든지 수정할 수 있어야 한다. (특히 닉네임 표기 수정 시, 접속 URL과의 연동도 고려됨을 명시)

### [필수] 링크 관리 (인라인 편집) 및 파비콘
* **단일 링크 추가:** 타이틀(Title)과 대상 URL을 입력하여 목록에 추가한다.
* **파비콘(Favicon) 아이콘 노출:** **Google Favicon API(`https://www.google.com/s2/favicons`)** 를 직접 활용하여 입력된 대상 URL의 도메인에서 빠르고 일관되게 공식 아이콘을 추출해 표시한다.
* **인라인 편집(Inline Edit) 및 삭제:** 링크 내용을 수정해야 할 때 별도의 모달(Modal)이나 새 페이지 창을 띄우지 않고, 현재 리스트의 **텍스트 영역을 직접 클릭해 즉시수정(Inline Edit)할 수 있는 직관적인 UI**를 제공한다. 변경이 완료된 링크 외 구버전 등 불필요한 항목은 삭제할 수 있다.

### [필수] 퍼블릭 페이지 구축 방안
* UI는 **shadcn/ui**의 컴포넌트를 베이스로 활용해 매우 모던하고 군더더기 없는 레이아웃으로 설계된다.
* 데스크탑, 태블릿, 모바일 등 접속 환경의 너비에 관계없이 컴포넌트가 적절하게 배열되는 **반응형 웹(Responsive Web)** 기술을 완벽하게 지원한다.
