# 마이링크 (MyLink) 사용자 시나리오 (User Scenarios)

## 1. 어드민 (관리자 및 소유자) 관점

### 1.1 계정 가입 및 로그인 (Onboarding)
* 사용자는 **서비스를 이용하기 위해** 메인 화면에서 **'구글 계정으로 로그인' 버튼을 클릭한다.**
* 사용자는 **처음 가입 시 본인만의 고유 접속 주소를 확보하기 위해** 온보딩 메뉴에서 URL 경로로 사용될 **닉네임(displayName)을 입력하고 중복 확인 후 가입을 한다.**
* 사용자는 **다른 기기에서 데이터 연동을 확보하기 위해** 동일한 **구글 계정으로 다시 로그인한다.**

### 1.2 프로필 관리 (마이페이지)
* 사용자는 **방문자에게 본인을 시각적으로 알리기 위해** 프로필 영역을 클릭하여 **자신의 사진(Avatar) 이미지를 컴퓨터 혹은 스마트폰에서 업로드한다.**
* 사용자는 **자신의 새로운 활동명이나 브랜드를 반영하기 위해** 표시 이름(displayName) 부분을 클릭하여 **새로운 닉네임을 입력하고 저장한다.**
* 사용자는 **방문자에게 간단한 환영 인사나 현재 상태를 알리기 위해** 자기소개(Bio) 영역을 클릭해 **간결한 텍스트(마크다운 지원)를 작성하고 저장한다.**

### 1.3 링크 추가 로직
* 사용자는 **새로운 포트폴리오나 웹사이트를 방문자에게 공유하기 위해** 마이페이지의 **'새 링크 추가' 버튼을 클릭한다.**
* 사용자는 **링크의 목적을 직관적으로 알리기 위해** 생성된 항목의 타이틀(Title)란에 **버튼에 가시적으로 표시될 콘텐츠 이름 텍스트를 입력한다.**
* 사용자는 **방문자를 정확한 목적지로 안내하기 위해** URL 입력란에 **이동할 대상 주소(URL)를 복사 혹은 타이핑하여 붙여넣기 한다.**
* 사용자는 **올바른 웹페이지로 유도하기 위해** `http://` 등이 누락되었을 때 나타나는 **"올바른 URL 형식을 입력해주세요"라는 유효성 경고 메시지를 확인하고 정상적인 구조로 수정한다.**
* 사용자는 **버튼 좌측에 대상 웹사이트의 아이콘이 제대로 연동되었는지 알아보기 위해** 저장을 완료하고 구글 API를 통해 실시간으로 렌더링되는 **파비콘(Favicon) 결과물을 확인한다.**

### 1.4 링크 수정 (인라인 편집)
* 사용자는 **기존에 등록한 링크 제목이나 목적지가 변경되었을 때 이를 신속하게 갱신하기 위해** 리스트에서 편집하려는 **텍스트 영역(Title 또는 URL)을 직접 클릭한다.**
* 사용자는 **불필요하게 별도의 모달창으로 넘어가는 번거로움을 피하기 위해** 해당 위치에 즉시 활성화된 입력 필드에서 **원하는 글자를 바로 변경하는 인라인(Inline) 편집을 수행한다.**
* 사용자는 **수정된 내용을 즉시 반영하기 위해** 유효성이 검증된 상태에서 키보드의 **Enter 키를 누르거나 마우스로 화면의 빈 공간(Blur)을 클릭한다.**

### 1.5 링크 삭제
* 사용자는 **기한이 지난 이벤트 링크나 더 이상 유효하지 않은 채널을 방문자 화면에서 숨기기 위해** 해당 링크 항목에 위치한 **'휴지통(삭제) 아이콘'을 클릭한다.**
* 사용자는 **버튼 조작 실수로 인해 링크 데이터가 날아가는 것을 예방하기 위해** 화면에 팝업된 "정말 삭제하시겠습니까?" 대화창에서 내용을 점검한 후 **'최종 삭제'를 눌러 확정한다.**

### 1.6 서비스 종료
* 사용자는 **공용 PC 환경 등에서 접근 보안을 유지하기 위해** 설정(또는 헤더) 메뉴에 있는 **'로그아웃' 버튼을 클릭한다.**

## 2. 방문자 (Visitor) 관점

### 2.1 사이트 정상 접속
* 사용자는 **크리에이터의 모든 소셜 미디어 및 포트폴리오를 탐색하기 위해** 크리에이터가 외부에 공유한 **마이링크 전용 URL(`mylink.com/닉네임`)을 클릭한다.**
* 사용자는 **자신이 현재 사용하는 기기에 구애받지 않고 컨텐츠를 열람하기 위해** 모바일, 태블릿, PC 등 다양한 환경의 웹 브라우저에서 **페이지에 반응형으로 맞추어진 UI 화면을 스크롤한다.**

### 2.2 프로필 확인 및 링크 이동
* 사용자는 **자신이 찾던 인물의 페이지가 맞는지 검증하기 위해** 화면 최상단에 렌더링된 **프로필 이미지, 닉네임, 그리고 바이오(Bio) 소개글을 읽는다.**
* 사용자는 **크리에이터가 운영하는 유튜브나 다른 외부 서비스로 곧바로 이동하기 위해** 직관적인 파비콘 아이콘이 포함된 **해당 링크 버튼 자체를 직접 클릭(터치)한다.**

### 2.3 잘못된 경로 (404 예외) 접속 처리
* 사용자는 **오타가 났거나 계정이 삭제되어 유효하지 않은 주소(`/잘못된닉네임`)를 클릭했을 때의 혼란을 방지하기 위해** "존재하지 않는 페이지입니다"라는 에러 텍스트 화면을 1차로 인지하고 **서비스의 메인 홈(마이링크 생성 유도 화면)으로 돌아가기 버튼을 클릭한다.**
